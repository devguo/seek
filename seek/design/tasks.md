### 游戏服务器框架开发任务

---

#### 第一阶段：公共基础 + dirsvr + netbus

##### 1.1 公共基础框架 (所有服务共用)

- [ ] 网络 IO 层: epoll/kqueue 事件循环, 支持 TCP/UDP/KCP/WebSocket 按配置开启
- [ ] 消息队列: 收包 → 按 playerId hash → 逻辑线程队列 → 写队列 → 发包, 保证同 playerId 有序
- [ ] 线程池: IO 线程池 + 逻辑线程池, 可配置线程数, playerId 绑核
- [ ] Protobuf 编解码: 统一消息头(uid/msgType/seq/version), 消息体 protobuf 3
- [ ] RPC 框架: 同步(协程)+异步(Future/Promise), 超时/重试/熔断
- [ ] UUID 生成器: 统一 ID 生成, 防止多实例 ID 重复
- [ ] 配置加载: 启动加载 + reload 热更, 配置文件格式(JSON/Lua)
- [ ] 日志系统: 结构化 JSON, 包含 uid/服务名/时间戳/级别
- [ ] 指标采集: Prometheus 指标导出(QPS/延迟/错误率/连接数/CPU/内存)
- [ ] 生命周期接口: 统一启动/停止/维护流程
- [ ] 目录服务客户端: 向 dirsvr 注册、心跳维持、服务列表订阅

##### 1.2 dirsvr (目录服务)

- [ ] 服务注册 API: 接收服务注册请求(类型/ID/IP/Port/权重)
- [ ] 心跳检测: 3s 间隔, 10s 超时标记 UNHEALTHY, 30s 标记 OFFLINE
- [ ] 服务列表管理: 内存为主, 落地本地文件用于重启恢复
- [ ] 变更推送: 服务列表变更时通知订阅者(netbus)
- [ ] 健康探测: 主动 TCP 连接检查
- [ ] 主备选举: Redis 分布式锁或简单选举, 主挂后备自动接管
- [ ] 网关列表查询: HTTP 接口, 客户端登录前获取可用网关列表
- [ ] 维护标记: 下发维护标记, 网关据此拒绝非白名单连接

##### 1.3 netbus (消息总线, 同一套代码多实例, 配置区分 NB1/NB2/NB3)

- [ ] 路由表管理: 从 dirsvr 拉取/订阅, 实时感知服务上下线
- [ ] 消息路由: 按 playerId hash / 消息类型 / 服务器类型 / 实例 ID 路由
- [ ] netbus 互联: NB1 ↔ NB2 ↔ NB3 网状连接, 跨服消息转发
- [ ] 广播/多播: 按服务器类型组播
- [ ] 消息可靠性: at-least-once 投递, 关键消息 ACK
- [ ] 背压处理: 下游积压时限流, 返回错误码
- [ ] 消息队列: 按 playerId 保证有序投递
- [ ] 下游健康跟踪: 不可用实例自动摘除
- [ ] NB3 副本负载均衡: 维护 raidsvr 副本数量, 新建副本分配到负载最低实例
- [ ] NB3 Token 签发: HMAC 签名, 绑定 playerId+raidId, 活动续期
- [ ] NB3 主备切换: 备接管后从 dirsvr 重建负载表

---

#### 第二阶段：网关 + gamesvr + dbsvr

##### 2.1 网关 (gateway)

- [ ] 客户端连接管理: TCP/UDP(KCP)/WebSocket 多协议接入
- [ ] KCP 可靠 UDP: 集成 KCP 协议栈
- [ ] 握手/心跳/断线重连
- [ ] 消息签名校验: HMAC-SHA256(消息体+timestamp+seq, 客户端密钥)
- [ ] 防重放: seq 递增校验 + timestamp 窗口
- [ ] 弱网切换: 检测 RTT 超阈值 → 建议客户端切 UDP(KCP)
- [ ] 客户端版本检查: 从配置读取最低版本号, 低版本拒绝连接
- [ ] 单 IP 频率限制: 连接数 + 消息频率
- [ ] 协议异常包检测与丢弃
- [ ] 直连 gamesvr: 服内消息直接转发, 不经过 netbus
- [ ] 连接 netbus: 需要转发的消息(排行榜/匹配等)走 netbus
- [ ] 副本期间双连接支持: 保持网关连接, 处理非战斗消息
- [ ] 白名单模式: 维护阶段只允许白名单 IP/账号连接

##### 2.2 gamesvr (游戏服, 与 raidsvr 共用代码, game_mode 启动)

- [ ] Player 对象管理: playerId hash 分配, 生命周期(preLoad→load→preLogin→login→afterLogin→...→logout→save)
- [ ] PlayerModule 基类: 定义 preInit/init/inited/preLoad/load/afterLoad/preLogin/login/afterLogin/preSave/save/afterSave/logout 虚接口
- [ ] 玩家模块: 背包/装备/商店/任务/社交/邮件(各继承 PlayerModule)
- [ ] Redis 数据读写: 玩家 KV 数据(道具/装备/背包等)的存取
- [ ] 世界数据管理: 大地图/公共场景按区域分片
- [ ] 视野同步线程: 独立的视野计算线程
- [ ] 网关直连接收: 处理网关直连转发的服内消息
- [ ] netbus 连接: 跨服消息 + 共享服务调用
- [ ] 跨服副本快照: 进入副本时记录 `{playerId, raidId, 带入道具, 初始属性}`
- [ ] 跨服副本流水: 接收 RaidSvr 异步消耗流水, 结算后应用回写
- [ ] 跨服副本宕机恢复: 快照 + 已收流水恢复玩家数据
- [ ] 玩家转服: 数据从源 gamesvr 迁移到目标 gamesvr
- [ ] 操作流水记录: 记录玩家操作流水(30 天, 存 PG)
- [ ] GM 指令处理: 查询/下发/封号/强制下线等

##### 2.3 dbsvr (数据库服)

- [ ] Redis ↔ PG 批量同步: 玩家 KV 数据定时 30s 批量写 PG
- [ ] 排行榜同步: Redis Sorted Set 定时每小时同步 PG
- [ ] 邮件/充值实时写入: 直接写 PG, 不经过 Redis
- [ ] 失败补偿: PG 写入失败入失败队列, 指数退避重试, 超阈值告警
- [ ] Redis AOF 持久化: Redis 重启自动恢复
- [ ] 写冲突防护: playerId 绑定固定存储线程
- [ ] PG 分表: player_data_00 ~ player_data_99

---

#### 第三阶段：raidsvr + matchsvr

##### 3.1 raidsvr (副本服, 与 gamesvr 共用代码, raid_mode 启动)

- [ ] 客户端接入: TCP/UDP(KCP)/WebSocket 多协议, 复用 IO 层
- [ ] 客户端消息校验: 签名/防重放, 复用网关校验模块
- [ ] Token 校验: HMAC 验证, 活动续期机制
- [ ] 战斗逻辑引擎: 移动/技能/伤害/Buff 计算, C++ 实现
- [ ] 副本实例生命周期: 创建 → 等待队员 → 战斗中 → 结算 → 销毁
- [ ] 单人副本: GameSvr → NB3 → RaidSvr 创建, 客户端直连
- [ ] 组队跨服副本: 批量 token, 全部到齐后开始, 超时 15s 未齐取消
- [ ] 副本战斗消息: Client ↔ RaidSvr 零跳直达
- [ ] 断线重连: 保留副本状态 15s, 原 token 重连, 超时结算退出
- [ ] 异步 RPC GameSvr: 拉取战斗属性, 发送消耗流水, 回写结算结果
- [ ] 副本结算: 清算结果, 通知所有队员退出
- [ ] NB3 连接: 副本管理/负载上报

##### 3.2 matchsvr (匹配服)

- [ ] 匹配队列管理: 加入/离开/超时处理
- [ ] 匹配算法: 按实力/积分/延迟匹配, 多池(1v1/3v3/大乱斗)
- [ ] 玩家信息获取: gamesvr 加入匹配队列时上报积分/段位
- [ ] 超时放宽: 匹配时间越长条件越宽松
- [ ] 结果通知: 匹配成功通知队长 gamesvr, 由 gamesvr 创建副本
